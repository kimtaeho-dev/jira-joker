# Jira 연동 도메인 - 기획서

> 최종 갱신: 2026-03-29

## 도메인 개요

Jira Cloud 및 Server·DC 인스턴스에 접속하여 Planning Poker 세션에 사용할 이슈 목록을 가져오는 역할을 담당한다. 사용자는 인증 정보를 입력하고, Epic을 검색하고, 하위 이슈를 불러와 포커 세션을 시작할 수 있다.

---

## Jira 인증 검증

### 이 기능은 무엇인가
사용자가 입력한 Jira 접속 정보(도메인, 이메일, 토큰)가 유효한지 실제 Jira 서버에 확인하는 기능이다. Cloud와 Server·DC 두 가지 인증 방식을 지원하며, 사용자가 선택한 모드에 맞게 필요한 정보를 검증한다.

### 사용자 시나리오
1. 사용자가 Cloud 또는 Server·DC 인증 모드를 선택한다
2. Cloud: 도메인, 이메일, API 토큰을 입력한다 / Server·DC: 도메인, Personal Access Token을 입력한다
3. "다음" 버튼을 클릭하면 시스템이 Jira 서버에 인증을 검증한다
4. 성공하면 인증 정보가 브라우저에 저장되고 다음 단계로 진행된다
5. 실패하면 오류 메시지가 표시되고 현재 단계에 머문다

### 왜 이렇게 동작하는가
- **이중 인증 모드 지원**: Cloud(SaaS)와 Server·DC(자체 호스팅)의 인증 방식이 다르므로 두 모드를 모두 지원하여 다양한 환경의 팀이 사용할 수 있다
- **서버 무저장 원칙**: 인증 정보를 서버에 보관하지 않고 매 요청마다 헤더로 전달하여 보안 리스크를 최소화한다
- **도메인 자동 정규화**: 사용자가 프로토콜이나 슬래시 형식을 정확히 몰라도 입력 편의를 제공한다

### 주요 정책
- POL-001: Cloud 모드에서 도메인·이메일·토큰 세 가지 필수 입력 (상세: POLICIES.md)
- POL-002: Server·DC 모드에서 도메인·토큰 두 가지 필수 입력 (상세: POLICIES.md)
- POL-005: 이메일 유무로 API v3/v2 및 인증 방식 자동 분기 (상세: POLICIES.md)
- POL-007: 서버는 인증 정보를 보관하지 않음 (상세: POLICIES.md)

### 에러 상황
- ERR-001: 모든 필드를 입력해주세요 — 누락된 필드를 모두 입력한 뒤 재시도
- ERR-002: Jira 인증 실패 — 도메인, 이메일, 토큰을 확인하고 재입력. 만료된 토큰은 재발급
- ERR-003: Jira 연결 실패 — 네트워크 상태를 확인하고 재시도

### 관련 용어
- Jira Cloud: Atlassian SaaS Jira 서비스 (상세: GLOSSARY.md)
- Jira Server·DC: 자체 호스팅 Jira (상세: GLOSSARY.md)
- API 토큰: Cloud 전용 인증 토큰 (상세: GLOSSARY.md)
- PAT: Server·DC 전용 인증 토큰 (상세: GLOSSARY.md)

## 인증 정보 캐싱

### 이 기능은 무엇인가
인증에 성공한 Jira 접속 정보와 닉네임을 브라우저에 저장하여, 다음 방문 시 입력 폼을 자동으로 채워주는 편의 기능이다. 저장된 정보는 사용자가 언제든 삭제할 수 있다.

### 사용자 시나리오
1. 인증 검증이 성공하면 시스템이 인증 정보를 자동으로 브라우저에 저장한다
2. 닉네임 입력 후 다음 단계로 진행하면 닉네임도 기존 캐시에 추가된다
3. 다음 방문 시 저장된 정보로 폼이 자동 채워진다
4. 사용자가 "저장된 정보 삭제" 버튼을 클릭하면 캐시가 제거되고 폼이 초기화된다

### 왜 이렇게 동작하는가
- **인증 성공 시에만 저장**: 잘못된 자격증명이 캐싱되어 다음 방문에 혼란을 주지 않도록 한다
- **닉네임 병합 저장**: 인증 정보와 닉네임을 한 곳에서 관리하여 삭제 시 한번에 정리된다
- **평문 저장**: 암호화 없이 저장되므로 공용 기기에서는 삭제가 권장된다

### 주요 정책
- POL-004: 인증 성공 시에만 캐싱 (상세: POLICIES.md)
- POL-009: 닉네임은 기존 자격증명에 병합 저장 (상세: POLICIES.md)
- POL-010: 삭제 시 폼 전체 초기화 (상세: POLICIES.md)
- POL-011: 인증 정보 평문 저장 (상세: POLICIES.md)

### 관련 용어
- 자격증명 캐시: 브라우저 로컬 저장소에 보관되는 인증 정보 (상세: GLOSSARY.md)

## Epic 검색

### 이 기능은 무엇인가
사용자가 입력한 Epic Key로 Jira에서 Epic을 검색하고, 해당 이슈가 실제 Epic 타입인지 검증하는 기능이다. Epic이 유효하면 자동으로 하위 이슈 목록 조회가 연속 실행된다.

### 사용자 시나리오
1. 사용자가 Epic Key(예: PROJ-42)를 입력한다
2. "검색" 버튼을 클릭하면 시스템이 Jira에서 해당 이슈를 조회한다
3. Epic 타입이 확인되면 Epic 정보(키, 제목)가 미리보기로 표시된다
4. 자동으로 하위 이슈 목록 조회가 시작된다
5. Epic 타입이 아니면 오류 메시지가 표시된다

### 왜 이렇게 동작하는가
- **Epic 타입 강제**: Planning Poker 세션의 범위를 Epic 단위로 정의하여, Story나 Task를 잘못 입력하는 실수를 방지한다
- **3가지 이름 인식**: 'epic', '에픽', '큰틀' — 다양한 Jira 설정 환경을 지원한다
- **자동 연속 조회**: Epic 확인 후 별도 버튼 없이 이슈 목록이 자동으로 불러와져 사용자의 조작 단계를 줄인다

### 주요 정책
- POL-012: Epic Key 미입력 시 검색 비활성 (상세: POLICIES.md)
- POL-013: Epic 타입이 아니면 오류 반환 (상세: POLICIES.md)
- POL-014: Epic Key 변경 시 이전 결과 자동 초기화 (상세: POLICIES.md)
- POL-015: Epic 확인 성공 후 하위 이슈 자동 연속 조회 (상세: POLICIES.md)

### 에러 상황
- ERR-004: Epic을 찾을 수 없음 — Epic Key가 올바른지, 접근 권한이 있는지 확인
- ERR-005: Epic 타입 불일치 — Jira에서 Epic 타입의 이슈 키를 확인하여 입력
- ERR-006: Epic 검색 실패 — 입력값 확인 후 재시도

### 관련 용어
- Epic: Jira 이슈 계층에서 세션 범위를 정의하는 상위 단위 (상세: GLOSSARY.md)
- Epic Key: 프로젝트 접두사와 이슈 번호로 구성된 식별자 (상세: GLOSSARY.md)

## 이슈 목록 조회

### 이 기능은 무엇인가
유효한 Epic의 하위 이슈(Story, Task, Bug)를 Jira에서 가져와 포커 세션의 투표 대상으로 제공하는 기능이다. Cloud와 Server·DC의 조회 방식 차이를 자동으로 처리하며, 이슈 설명 형식도 통일된 텍스트로 변환한다.

### 사용자 시나리오
1. Epic 검색이 성공하면 시스템이 자동으로 하위 이슈를 조회한다
2. 이슈 목록(최대 100건)이 표시되며 각 이슈의 키, 제목, 기존 Story Points가 보인다
3. 이슈가 1건 이상이면 "방 만들기" 버튼이 활성화된다
4. 이슈가 0건이면 방 만들기가 차단된다

### 왜 이렇게 동작하는가
- **Cloud·Server 자동 분기**: Cloud는 부모-자식 관계로, Server·DC는 Epic Link 필드로 하위 이슈를 조회한다. 사용자가 조회 방식을 신경 쓸 필요가 없다
- **100건 제한**: 지나치게 많은 이슈를 한 세션에 넣으면 진행이 비효율적이므로 현실적인 범위로 제한한다
- **ADF 자동 변환**: Cloud의 구조화된 문서 형식을 평문으로 변환하여 포커 테이블에서 일관된 텍스트로 표시한다

### 주요 정책
- POL-016: 이슈 0건이면 방 만들기 차단 (상세: POLICIES.md)
- POL-017: 최대 100건 조회 (상세: POLICIES.md)
- POL-018: Story Points 필드 식별 — Cloud 커스텀 필드 10016 (상세: POLICIES.md)
- POL-019: Cloud·Server·DC 조회 방식 자동 분기 (상세: POLICIES.md)
- POL-020: Cloud 이슈 설명 평문 자동 변환 (상세: POLICIES.md)

### 에러 상황
- ERR-007: 이슈 목록 조회 실패 — Jira 접근 권한을 확인
- ERR-008: 서버 내부 오류 — 네트워크 상태와 도메인 주소를 확인하고 재시도

### 관련 용어
- Story Points: 이슈의 작업 규모를 나타내는 상대적 수치 (상세: GLOSSARY.md)
- ADF: Jira Cloud API v3의 구조화 문서 형식 (상세: GLOSSARY.md)
- JQL: Jira 이슈 검색 쿼리 언어 (상세: GLOSSARY.md)
